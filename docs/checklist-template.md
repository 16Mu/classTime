### 1.1 URL 可访问性

#### 检测工具
```bash
node scripts/testing/accessibility/url-checker.js --school <schoolId>
```

#### 检查项

- [ ] **loginUrl 可访问（HTTP 200/301/302）**

  | 指标 | 结果 |
  |------|------|
  | 状态码 | `____` |
  | 响应时间 | `____` ms |
  | Content-Type | `____` |
  | 截图路径 | `____` |

  **判定标准**: 状态码为 200/301/302，响应时间 < 10 秒

- [ ] **scheduleUrl 可访问（HTTP 200/301/302）**

  | 指标 | 结果 |
  |------|------|
  | 状态码 | `____` |
  | 响应时间 | `____` ms |
  | Content-Type | `____` |

  **备注**: 若 scheduleUrl 为空（部分系统需要登录后动态获取），标注"无需预检测"

- [ ] **SSL 证书有效**（仅 HTTPS 站点）

  | 指标 | 结果 |
  |------|------|
  | 证书状态 | `____` (有效 / 已过期 / 自签名) |
  | 过期时间 | `____` |
  | 剩余天数 | `____` 天 |
  | 颁发机构 | `____` |

  **判定标准**: 证书有效且剩余 > 30 天，或为受信任的自签名证书

#### 常见问题处理

| 问题现象 | 可能原因 | 处理建议 |
|----------|----------|----------|
| ECONNREFUSED | 服务器未启动或端口错误 | 联系学校确认服务状态 |
| ENOTFOUND | DNS 解析失败 | 检查 URL 是否正确 |
| ETIMEDOUT | 网络超时或防火墙拦截 | 检查网络环境，考虑代理 |
| CERT_HAS_EXPIRED | SSL 证书过期 | 联系学校更新证书 |
